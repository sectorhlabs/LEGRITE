La historia

Mi primer proyecto con IA

Mi padre trabaja en una empresa de instalaciones térmicas. Cada semana tiene que generar decenas de documentos oficiales — certificados, memorias técnicas, solicitudes — para legalizar las instalaciones de aire acondicionado, calefacción y ventilación que realizan.


El proceso era manual y repetitivo: abrir un PDF, buscar los campos, copiar datos del instalador, del cliente, de la máquina instalada... Un trabajo que podía llevar horas cada semana.


LEGRITE nació para cambiar eso. Una aplicación de escritorio hecha en Java que carga los datos una sola vez y genera todos los PDFs oficiales en un click. Fue el primer proyecto real que construí usando inteligencia artificial como asistente de desarrollo.

"Lo más valioso no fue el código, fue ver cómo una herramienta podía ahorrarle a mi padre horas de trabajo repetitivo cada semana."
Tecnologías

Stack técnico

Aplicación de escritorio con interfaz gráfica (Swing + FlatLaf) y modo terminal (CLI con JSON) para automatización. El motor principal usa Apache PDFBox para rellenar formularios AcroForm.

Cargar datos

02
Rellenar formulario
Datos del titular, dirección de la instalación, características de hasta 4 máquinas en locales distintos.
03
Generar PDFs
El motor PDFBox rellena automáticamente los campos AcroForm de las plantillas oficiales con los datos introducidos.
04
Entregar
Todos los certificados y memorias listos para firma y presentación telemática en el organismo correspondiente.
Modo terminal

CLI para automatización

Además de la interfaz gráfica, LEGRITE se puede usar desde terminal pasando un JSON. Esto lo hace compatible con agentes de IA y permite integrarlo en flujos automatizados.

# Modo interactivo: lista todos los campos disponibles
java -jar LEGRITE-cli.jar --campos

# Modo batch: genera PDFs desde un JSON
java -jar LEGRITE-cli.jar --json datos.json --plantillas Modelos/ --salida salida/

# Desde stdin (compatible con pipes y agentes IA)
echo '{"TI1":"Juan","TI2":"Garcia"}' | java -jar LEGRITE-cli.jar --json - --plantillas Modelos/ --salida salida/
El dato curioso

Decompilado y reconstruido

La versión original de LEGRITE se perdió al no estar versionada con Git. El JAR compilado fue decompilado a código fuente con CFR 0.152 y reconstruido desde ahí, añadiendo nuevas funcionalidades como el motor PDFBox real, el modo CLI, FlatLaf y el subproyecto del Catastro.


Es un ejemplo de cómo incluso cuando las cosas salen mal (perder el código fuente), con las herramientas adecuadas se puede recuperar y mejorar el proyecto.
